# AI 자동화 개발 계획

이 폴더는 EPL X 자동 수집/AI 검수 MVP 개발의 기준 문서다.

모든 개발 세션은 아래 순서로 시작한다.

1. `99-progress.md`를 읽는다.
2. `현재 커서`에 적힌 작업 ID를 확인한다.
3. 해당 작업이 매핑된 상세 문서를 읽는다.
4. `99-progress.md`에 적힌 주요 파일만 확인한다.
5. 해당 작업 하나만 구현한다.
6. 완료 조건을 통과하면 `99-progress.md`의 체크박스를 갱신한다.
7. `현재 커서`를 다음 미완료 작업으로 이동한다.

기억에 의존해서 다음 작업을 고르지 않는다. 사용자가 명시적으로 우선순위를 바꾸지 않는 한 `99-progress.md`의 첫 번째 미완료 작업부터 진행한다.

## 단계 순서

| 단계 | 문서 | 목표 |
|---|---|---|
| P0 | `00-master-roadmap.md` | 제품 목표, 아키텍처, 단계 규칙 정렬 |
| P1 | `01-content-contract.md` | AI 분류와 한국어 브리핑 출력 계약 확정 |
| P2 | `02-data-model.md` | Supabase 스키마, seed, 마이그레이션 확정 |
| P3 | `03-ingestion-pipeline.md` | X 수집, 중복 제거, AI 분류, 라우팅 |
| P3 사전작업 | `07-p3-preflight-guide.md` | X/Supabase/OpenAI/Slack/Vercel 실제 연동 준비 |
| P4 | `04-admin-review.md` | 관리자 검수 큐와 승인/반려 플로우 |
| P5 | `05-feed-publishing.md` | 승인된 콘텐츠를 Reax 피드에 반영 |
| P6 | `06-deployment-ops.md` | 배포, 스케줄링, 모니터링, MVP 검증 |

## 작업 규칙

- 각 작업 상태는 `[ ]`, `[~]`, `[x]`, `[!]` 중 하나로만 표시한다.
- 완료 조건을 통과하지 않은 작업은 완료 처리하지 않는다.
- 공개 API, DB 스키마, 환경변수, AI 출력 계약, 관리자 플로우가 바뀌면 관련 문서를 같은 변경 안에서 갱신한다.
- 실데이터 검증 전까지 기존 mock fallback은 유지한다.
- 자동 발행은 보수적으로 처리한다. 루머, 애매한 글, 미디어 중심 글, 추상적인 글은 검수 큐로 보낸다.
